# Layout Types Create Avatar Rhythm

**Source:** Claude iOS video planning + Kimi video analysis (2025-12-03)
**Validated:** Kimi video uses this pattern; our Claude iOS video adopted it

## The Insight

Define 2-3 layout types and alternate between them to create rhythm. The key variable is **avatar presence** - appearing for emotional connection, disappearing to let UI breathe.

## Why It Works

- **Avatar visible** = human connection, trust, emotional beats
- **Avatar hidden** = UI is the star, proof/demonstration, let visuals breathe
- **Full avatar** = maximum presence for punchlines and CTAs

Alternating creates pacing variety and signals to viewers when to connect emotionally vs absorb information.

## The Three Layout Types

### Type A: With Avatar (bottom 48%)
```
┌─────────────────┐
│  Top text       │
│  ┌───────────┐  │
│  │ Content   │  │  ← Compressed to top 52%
│  └───────────┘  │
│  Caption        │
├─────────────────┤
│    AVATAR       │  ← Bottom 48%
└─────────────────┘
```
**Use for:** Hook, pivots, emotional moments

### Type B: Full Canvas (no avatar)
```
┌─────────────────┐
│  Top text       │
│                 │
│  ┌───────────┐  │
│  │  LARGE    │  │  ← Content uses full height
│  │  CONTENT  │  │
│  └───────────┘  │
│                 │
│  Caption        │
└─────────────────┘
```
**Use for:** UI demos, proof, showcasing features

### Type C: Full Avatar
```
┌─────────────────┐
│  ┌───────────┐  │  ← Overlay card (optional)
│  │ Text      │  │
│  └───────────┘  │
│░░░░░░░░░░░░░░░░░│
│░░░ AVATAR ░░░░░░│  ← Full bleed
│░░░░░░░░░░░░░░░░░│
│  Caption        │
└─────────────────┘
```
**Use for:** Punchlines, CTA, strong closes

## How to Apply

1. Assign each story beat a layout type
2. Check the sequence - does avatar appear/disappear at meaningful moments?
3. Ensure CTA and closing are Type C (full avatar for human connection)

## Example from Claude iOS Video

```
KF1  [A] Hook - avatar speaks
KF2  [B] Siri fails - UI showcase
KF3  [B] Memory fails - UI showcase
KF4  [A] "But" pivot - avatar returns for connection
KF5  [B] Claude hero - logo moment
KF6-13 [B] Demos - let UI prove it works
KF14 [B] Memory - visualization
KF15 [C] "Finally" - full avatar for punchline
KF16 [C] CTA - full avatar
KF17 [C] Last Dab - strong close
```

The rhythm: Avatar IN → OUT (proof) → IN (pivot) → OUT (demos) → FULL (payoff)
